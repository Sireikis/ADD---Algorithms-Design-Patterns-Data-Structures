Binary Search

Summary

Binary Search expects a sorted array or other sorted iterable collection. It searches for the given element, called the target, by repeatedly dividing the array in half. Binary Search first determines the low and high indices of the array. In the beginning, these are equivalent to the first and last index, respectively. Then, while the low and high indices don't overlap, it finds the middle index (by adding low and high indices and dividing by two) and performs the following steps until the indices DO overlap, or the target is found:

  1. Check if the element at the middle index is the target, if so return the element and search is complete.
  2. If the target is less than that element, the high index becomes the middle index minus one and the while loop repeats.
  3. If the target is greater than that element, the low index becomes the middle index plus one and the while loop repeats.

Binary Search can be applied to tree structures, such as a Binary Search Tree. The structure of a Binary Search Tree follows the invariant "given a node, the left subtree will contain values less than it while the right subtree will contain values greater than it." Hence, instead of manipulating indices, we simply traverse nodes according to the given target until it is found or no path forward exists.
